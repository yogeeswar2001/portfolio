# Yogeeswar Senagapalli ‚Äî Portfolio

Personal portfolio website hosted on GitHub Pages, with an automated pipeline that parses a LaTeX resume PDF and keeps the page content in sync.

**Live site:** [yogeeswar2001.github.io/portfolio](https://yogeeswar2001.github.io/portfolio)

## Updating the resume

1. Make your changes in Overleaf
2. Download the compiled PDF: **Menu ‚Üí PDF ‚Üí Download**
3. Replace `resume.pdf` in the repo root
4. Push to `main`:
   ```bash
   git add resume.pdf
   git commit -m "update resume"
   git push origin main
   ```
5. The GitHub Action fires automatically, parses the PDF, updates `resume.json`, and GitHub Pages redeploys

---

## Updating static content

Some fields can't be parsed from the PDF (links, about text, cert badge URLs). Edit `resume_static.json` directly:

| Field | What it controls |
|---|---|
| `tagline` | Subtitle under your name in the hero section |
| `about` | The three paragraphs in the About Me section |
| `certifications` | Certification names + badge URLs in the Skills section |
| `project_links` | URLs injected into each project/patent entry |

The key in `project_links` must exactly match the project title as it appears in your LaTeX source.

## Testing locally

The page uses `fetch("resume.json")` which requires an HTTP server ‚Äî it won't work by just opening `index.html` in a browser.

```bash
# from the repo root
python3 -m http.server 8000
```

Then open [http://localhost:8000](http://localhost:8000).

To regenerate `resume.json` locally after editing the PDF or `resume_static.json`:

```bash
pip install pdfplumber
python3 scripts/parse_resume.py
```

## GitHub Actions setup

The workflow (`sync-resume.yml`) requires no secrets ‚Äî it only runs `parse_resume.py` which reads from local files in the repo.

To manually trigger a run without pushing a new PDF:

1. Go to **Actions** tab
2. Click **Sync Resume from Overleaf**
3. Click **Run workflow**
